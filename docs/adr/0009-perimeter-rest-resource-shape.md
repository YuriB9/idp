# ADR-0009: Форма REST-ресурсов периметра (проектно-скоупленные пути)

**Status:** Accepted
**Date:** 2026-06-18
**Change:** create-service-portal

## Context

ADR-0002 зафиксировал: внутри — gRPC, на периметре (портал ↔ API-шлюз) —
OpenAPI/https/json. В каркасе периметра был один заглушечный `GET /services`,
возвращавший `[]`. Для сквозного сценария «Создание сервиса» нужны доменные
операции периметра: запуск создания, чтение статуса, листинг. Требуется выбрать
устойчивую форму URL-ресурсов, согласованную с gRPC `projectsv1` и расширяемую на
будущие операции (владельцы, перенос, decommission), которые сейчас вне scope.

## Considered Options

1. **Проектно-скоупленные вложенные ресурсы**:
   `POST /projects/{project}/services`, `GET /projects/{project}/services/{name}`,
   `GET /projects/{project}/services` — владение проектом выражено в пути.
2. **Плоский ресурс с query-параметром**: `GET /services?project=...`,
   `POST /services` с `project` в теле — проще на старте, но владелец не выражен
   в иерархии и хуже масштабируется на под-операции.

## Decision

Принимается вариант 1: доменные операции периметра адресуются
проектно-скоупленными путями `/projects/{project}/services[/{name}]`. Каркасный
`GET /services` удаляется (**BREAKING**, потребителей не было). Формы тел/ответов
выводятся из gRPC-контракта `projectsv1` (id+status при создании; project/name/
status при чтении; keyset `page_size`/`page_token`/`next_page_token` при
листинге). OpenAPI остаётся единственным источником правды; gateway лишь маппит
REST↔gRPC, не раскрывая внутренних ошибок (ADR-0003), и маппит gRPC-коды в HTTP.

## Consequences

**Положительные:** иерархия отражает владение проектом; единый префикс для
будущих под-операций сервиса; естественное соответствие gRPC-вызовам каталога;
понятные 404/409/400 по ресурсу.

**Отрицательные:** **BREAKING** относительно каркасного `GET /services` (требует
перегенерации TS/zod, `gen:check`); чуть длиннее пути.

**Контракт:** изменения OpenAPI периметра, ломающие форму, помечаются BREAKING;
после правок обязателен `npm run gen` и зелёный `gen:check`.
