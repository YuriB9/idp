# ADR-0017: Дизайн-система и UI-архитектура портала

**Status:** Accepted
**Date:** 2026-06-21
**Change:** portal-ui-design-system

## Context

Портал `./web` (React 19 + Vite + TS + TanStack Query + react-router; формы
react-hook-form + zod; рантайм-валидация ответов zod `.parse`) вырос органически:
разделы собраны из разрозненных карточек/списков, действия — инлайновые формы,
навигация плоская, состояния loading/empty/error и сообщения об ошибках оформлены
по-разному, нет единой семантики статусов и критичности. Это рефакторинг только
фронтенда: контракт периметра (ADR-0009: keyset/opaque-курсоры
`next_page_token`/`next_cursor`, коды 400/403/404/409/422/503), proto и поведение
бэкенда не меняются; IAM-разделы (ADR-0014/0015/0016) сохраняют поведение и права.
Существующий слой примитивов — `web/src/components/ui/*` (shadcn-подход, `cva` +
Tailwind-токены), тема — CSS-переменные (oklch) + `@theme` с `ThemeProvider`
(ключ `idp-theme`, дефолт тёмная). Ограничения: не менять кодоген
(`web/src/api/*`, `*/openapi.yaml` — `gen:check` без diff), не понижать zod
3.25.76, не тянуть тяжёлые UI-киты, доступность (WCAG AA) — требование первого
класса.

## Decision

- **Дизайн-система = расширение существующих shadcn-примитивов**, а не параллельный
  слой. Новые примитивы (DataTable, Sidebar, Dialog/ConfirmDialog, Toast,
  PageHeader, бейджи статусов/критичности) добавляются в `components/ui` и рядом,
  переиспользуя `Button`/`Input`/`Card`/`Badge`. Это исключает дубли и две
  конкурирующие системы.
- **Источник токенов — CSS-переменные + Tailwind `@theme`.** Палитра состояний и
  семантические уровни критичности добавляются как токены поверх `:root`/`.dark`,
  не ломая `ThemeProvider`/FOUC-защиту и дефолт темы. Примитивы используют только
  токены.
- **Семантика статусов/критичности — единый компонент «цвет + иконка + текст»**
  (colorblind-safe), один источник правды маппинга `ServiceStatus` и уровней
  критичности; неизвестный статус деградирует в нейтральный бейдж.
- **DataTable — собственная лёгкая реализация** (семантический `<table>` с
  `role`/заголовками/`aria-sort`), а не TanStack Table: меньший бандл, полный
  контроль a11y/состояний и КУРСОРНОЙ (не offset) пагинации по ADR-0009 (курсор
  пробрасывается без интерпретации; пустой курсор → конец). Клиентская сортировка —
  там, где набор загружен целиком.
- **Модалки и тосты — собственные лёгкие примитивы** с focus-trap, ESC, возвратом
  фокуса, блокировкой скролла и ARIA, без тяжёлых UI-китов и без понижения zod;
  формы внутри — react-hook-form + zod, ответы — zod `.parse`. Деструктивные
  операции — через отдельный `ConfirmDialog`.
- **Единый UI-маппинг кодов периметра** (400/403/404/409/422/503 → стабильные
  сообщения) в общем модуле `web/src/lib`, переиспользуемый тостами и блоками
  ошибок; сырые внутренние ошибки наружу не идут. Консолидирует разрозненные
  `httpStatusOf`/`mutationMessage`/`catalogMessage`.
- **Состояние UI в localStorage** согласовано со схемой темы: `idp-sidebar`
  (свёрнуто/развёрнуто), `idp-density` (comfortable/compact) — безопасное чтение с
  валидацией по белому списку и деградацией к дефолту.
- **Доступность — требование первого класса**: WCAG AA контраст в обеих темах,
  клавиатура, видимый фокус, ARIA-роли для таблиц/меню/диалогов, статусы не только
  цветом, `prefers-reduced-motion`.
- **Рефактор инкрементальный** (страница за страницей) с зелёными `tsc --noEmit` и
  vitest на каждом шаге; контракт и кодоген не трогаются.

## Consequences

- Единый визуальный язык, плотность данных и иерархия критичности; консистентные
  состояния и сообщения об ошибках во всех разделах.
- Минимум новых зависимостей и контроль бандла/доступности ценой собственной
  поддержки примитивов (DataTable/Dialog/Toast) — покрывается vitest.
- Поведение и права (IAM-пикер, 403/503/«осиротевшие», optimistic-concurrency
  владельцев, предусловия decommission/transfer) сохраняются; риск регрессии
  закрывается тестами до/после.
- `gen:check` остаётся зелёным, т.к. контракт и сгенерированные файлы не меняются;
  выход на правку контракта — сигнал выхода за scope (отдельный change).
- Опирается на ADR-0009 (форма периметра, курсоры, коды ошибок) и не отменяет
  ADR-0014/0015/0016 (поведение IAM сохраняется).
