# ADR-0007: Инструмент миграций БД — goose

**Status:** Accepted
**Date:** 2026-06-18
**Change:** projects-catalog

## Context

Каталогу проектов нужна управляемая, версионированная и обратимая схема Postgres (таблица `services` и далее). Инструменты кодогена уже пинуются в изолированном модуле `./tools` (`GOWORK=off`). Нужен инструмент миграций, согласованный с этим подходом: без CGO, с обратимыми шагами, пригодный и для CLI-применения локально, и для будущего применения из integration-тестов.

## Decision

Используем **goose** (`github.com/pressly/goose/v3`):
- Пин в `./tools` (`GOWORK=off`), как и прочие инструменты.
- Миграции — `.sql`-файлы в `services/projects/migrations` с парами `-- +goose Up` / `-- +goose Down` (прямой и обратный шаг в одном файле).
- Применение — Makefile-таргет (`make migrate-projects`) с DSN из окружения; возможно программное применение через `embed.FS` в integration-тестах.

## Consequences

**Положительные:** один бинарь без CGO и без формата-драйвера в строке источника; обратимость рядом с прямой миграцией; нативная работа с Postgres/pgx; лёгкая встройка в тесты.

**Отрицательные:** ещё один инструмент в `./tools` (нужно пинить версию и поддерживать таргет); смешанный SQL-синтаксис с директивами goose в файлах миграций.

**Альтернатива (отклонена):** golang-migrate — раздельные up/down-файлы и драйвер в строке источника, больше церемонии для текущего объёма схемы.
