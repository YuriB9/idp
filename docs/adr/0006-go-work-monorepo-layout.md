# ADR-0006: Раскладка монорепо на go.work (модуль на сервис + изолированный tools)

**Status:** Accepted
**Date:** 2026-06-18
**Change:** foundation-and-pkg

## Context

MVP — монорепо на Go с общими пакетами и четырьмя сервисами (`gateway`, `idm`, `projects`, `devinfra-worker`), плюс сквозные тесты и инструменты кодогена/линтинга. Нужно решить раскладку модулей так, чтобы: CI давал честную матрицу по компонентам, графы зависимостей сервисов не смешивались, а версии инструментов (protoc/buf, golangci-lint, кодоген) не протекали в рантайм-зависимости сервисов. Урок прошлого проекта на близком стеке: общие пакеты копипастить нельзя, а tooling-зависимости в общем модуле засоряют граф.

## Considered Options

1. **Один Go-модуль на весь репозиторий.** Просто, но один общий граф зависимостей, нет естественной матрицы CI, tooling-зависимости тянутся всюду.
2. **`go.work` с модулем на компонент (`pkg`, каждый сервис, `tests/e2e`) + изолированный `tools` вне workspace.** Независимые `go.mod`, честная матрица CI, изоляция инструментов.
3. **Несколько отдельных репозиториев.** Чистая изоляция, но дорогая координация контрактов/кодогена на стадии MVP.

## Decision

Принимаем вариант 2. Монорепо на `go.work`:

- `./pkg` — общие пакеты (один модуль).
- `./services/{gateway,idm,projects,devinfra-worker}` — по модулю на сервис.
- `./tests/e2e` — модуль сквозных тестов.
- `./tools` — **вне** `go.work`, со своим `go.mod`, пинит версии инструментов; команды запускаются с `GOWORK=off`.
- Frontend — `./web` (вне Go-workspace).

CI строит матрицу по модулям `go.work`; `tools` используется только для кодогена/линтинга.

## Consequences

**Положительные:** честная матрица CI по компонентам; изолированные графы зависимостей сервисов; tooling-зависимости не протекают в сервисы; раскладка совпадает с целевыми границами (gRPC/Temporal), наполнение позже не требует переписывать ядро.

**Отрицательные:** дисциплина `go mod tidy` по каждому модулю (проверяется в CI через `git diff --exit-code`); чуть больше церемоний при добавлении зависимостей; нужно помнить про `GOWORK=off` для `tools`.
