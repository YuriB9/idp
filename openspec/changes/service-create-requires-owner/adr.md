# ADR (изменение: service-create-requires-owner)

Этот файл — индекс архитектурных решений данного изменения. Канонические ADR
лежат в `docs/adr/NNNN-kebab-title.md` (вне `openspec/`).

## Новые ADR

- **ADR-0023 — Обязательные владельцы при создании сервиса и полное установление
  владения в воркфлоу** (`docs/adr/0023-owners-required-at-service-creation.md`):
  `CreateServiceRequest` получает обязательное поле `owners` (proto+openapi,
  BREAKING); владельцы фиксируются в каталоге атомарно при вставке
  (`owners_version=1`), а воркфлоу создания устанавливает реальные роли владельца
  (GitLab members → Vault policies → IDM owner roles) с Saga-компенсацией
  (вариант B), переиспользуя activities `changeowners`.

## Реализуемые существующие ADR

- **ADR-0011 — Контракт владельцев и синхронизация ролей**: сквозной механизм
  синхронизации владельцев применяется уже при создании (`add=owners`).
- **ADR-0004 — guarded-CAS статусов**: транзакционная вставка записи с
  владельцами; best-effort `creating→failed` при сбое запуска.
- **ADR-0005 — Saga-откат**: компенсации шагов воркфлоу создания; IDM-роли после
  активации без молчаливого отката.
- **ADR-0008 — split воркфлоу**: запись каталога — источник правды; шаги после
  активации не откатывают созданное.
- **ADR-0009 — форма периметра**: синхронная валидация пустых владельцев → 400 без
  сырых деталей.
- **ADR-0016 — справочник субъектов IDM**: сознательно без существ. проверки
  владельцев при создании (консистентно с ChangeOwners).
- **ADR-0022 — единый прогресс**: обновление подписей `OPERATION_STEPS.create`.
