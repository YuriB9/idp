## Why

Фронтенд портала `./web` собран на Vite 6.0.7 — это мажор на два релиза старше
актуального (Vite 8.1.0). Чтобы оставаться на поддерживаемой ветке инструментов
(security-фиксы, совместимость с экосистемой плагинов, актуальный baseline
сборки) и не накапливать разрыв, который позже превратится в болезненную
многошаговую миграцию, нужно поднять сборку до Vite 8 СЕЙЧАС, пока поверхность
конфига маленькая (один `vite.config.ts`, встроенный блок vitest, без отдельных
конфигов и project references). Это инфраструктурно-инструментальное изменение
в рамках стека, зафиксированного ADR-0017 (React 19 + Vite) и раздела «Стек»
docs/IDP_MVP_План.md; смены бандлера/раннера НЕ происходит, поэтому новый ADR
не требуется.

## What Changes

- Поднять `vite` `6.0.7` → `8.1.0` (мажорный апгрейд через Vite 7).
- Поднять `@vitejs/plugin-react` `4.3.4` → `^6` (линейка 4.x несовместима с
  Vite 7/8; v6 имеет peer `vite: ^8`). **BREAKING** (на уровне dev-зависимости).
- `vitest` `4.1.9` ОСТАЁТСЯ как есть: его peer уже объявляет
  `vite: "^6.0.0 || ^7.0.0 || ^8.0.0"` — бамп не нужен; 154 теста, jsdom,
  `globals`, `setupFiles` сохраняются без изменений API.
- `@tailwindcss/vite` `4.3.1` и `tailwindcss` `4.3.1` ОСТАЮТСЯ: peer
  `@tailwindcss/vite` уже `vite: "^5.2.0 || ^6 || ^7 || ^8"`; стили не меняются.
- `jsdom 29.1.1` ОСТАЁТСЯ (актуальная версия).
- `vite.config.ts`: заменить ESM-несовместимый `__dirname` (риск под нативным
  ESM-загрузчиком конфигов Vite) на `import.meta.dirname`; сохранить alias
  `@`→`src`, dev-прокси `/api`→gateway (env `GATEWAY_URL`, порт 3000),
  плагины `react()` + `tailwindcss()` и встроенный блок `test` (vitest).
- `package-lock.json` пересобрать воспроизводимо (`npm ci` зелёный).
- Node baseline: Vite 8 требует Node 20.19+/22.12+; CI и локально — Node 24,
  требование удовлетворено. При необходимости зафиксировать `engines` в
  `package.json` (CI-матрица Node не меняется).
- Все комментарии в коде/конфигах — только на русском.

## Impact

- **Файлы:** `web/package.json`, `web/package-lock.json`, `web/vite.config.ts`;
  при необходимости `web/tsconfig.json`, `web/src/test/setup.ts`.
- **Dependabot (`.github/dependabot.yml`):** в npm-экосистеме `/web` добавлена
  группа `vite` (`vite`, `@vitejs/*`, `@tailwindcss/vite`, `vitest`), чтобы
  связанные peer-зависимости предлагались одним согласованным PR (устраняет
  повтор ERESOLVE из закрытых PR #64/#26).
- **CI (`.github/workflows/ci.yml`):** правка ТОЛЬКО при смене Node baseline
  (сейчас не требуется — Node 24 покрывает Vite 8). Джобы `web-test`
  (`tsc --noEmit` + vitest), `openapi-lint`, `codegen-check` должны остаться
  зелёными; Go-джобы не затрагиваются.
- **Поведение:** без изменений (dev-прокси, алиасы, Tailwind, 154 vitest-теста,
  кодоген OpenAPI).
- **Зависимости:** мажор `vite`, мажор `@vitejs/plugin-react`; прочие
  Vite-зависимые пакеты совместимы без бампа.
- **Откат/компенсации:** провизию ресурсов изменение НЕ затрагивает (нет
  Saga/Temporal/gRPC). Откат — ревертом PR (восстановление прежних версий и
  `package-lock.json`).
- **Границы сервисов (gRPC/Temporal):** не затрагиваются; контракт
  proto/OpenAPI, бэкенд и `tests/e2e` вне границ.
