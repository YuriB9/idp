# ADR (изменение: portal-iam-users-permissions-ux)

Этот файл — индекс архитектурных решений данного изменения. Канонические ADR
лежат в `docs/adr/NNNN-kebab-title.md` (вне `openspec/`).

## Реализуемые существующие ADR

Новых ADR это изменение НЕ создаёт: это чистый UX-рефактор фронтенда без новых
сервисных границ, контрактов периметра, транспорта или модели данных. Решения по
подаче форм/таблиц (модалка назначения роли, подсказки `action`/`resource` через
нативный `<datalist>`, локальный клиентский фильтр каталога прав поверх неизменного
`DataTable`) — применение UI-архитектуры, уже зафиксированной в ADR-0017
(см. `design.md`). Выделение фильтра в общий примитив `DataTable` сознательно
отложено и при будущей реализации потребует отдельного ADR.

- **ADR-0017 — Дизайн-система и UI-архитектура портала**: воплощается переносом
  назначения роли в `Dialog` за кнопкой `PageHeader.actions` (паттерн «таблица +
  кнопка → модалка», как на «Ролях»), подсказками доступных действий/ресурсов и
  локальным тулбаром фильтрации поверх существующих примитивов (`DataTable`,
  `Dialog`, `ConfirmDialog`, `Input`, `toast`, `PageHeader`) без их дублирования.
- **ADR-0009 — Форма REST-ресурса периметра**: сохраняется курсорная пагинация
  (`next_page_token`) на странице «Пользователи»; клиентский фильтр каталога прав
  не затрагивает форму ресурса и работает поверх загруженных данных.
- **ADR-0014/0015/0016 — IAM-админка (авторизация/чтение, динамический каталог,
  справочник субъектов)**: поведение прав (`read`/`manage` `iam:global`,
  `read iam:directory`), read-only системных прав, пикер с debounce и обработка
  403/503/осиротевших сохраняются без изменений; источник подсказок
  `action`/`resource` — производные от `listPermissions`, новых эндпоинтов нет.
