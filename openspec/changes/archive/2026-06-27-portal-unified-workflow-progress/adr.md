# ADR (изменение: portal-unified-workflow-progress)

Этот файл — индекс архитектурных решений данного изменения. Канонические ADR
лежат в `docs/adr/NNNN-kebab-title.md` (вне `openspec/`).

## Новые ADR

- **ADR-0022 — Единый ступенчатый прогресс операций в портале и источник «шагов
  воркфлоу»** (`docs/adr/0022-portal-unified-workflow-progress-source.md`): вводится
  единый презентационный примитив ступенчатого прогресса для всех операций над сервисом;
  источник шагов — вариант B (вывод из грубого статуса периметра на клиенте, без
  изменения контракта/кодогена); тост — только для синхронных ошибок запуска. Вариант A
  (per-activity шаги Temporal через расширение контракта) отвергнут для этой задачи и
  задокументирован как продолжение.

## Реализуемые существующие ADR

- **ADR-0017 — дизайн-система и UI-архитектура портала**: новый UI-примитив ступенчатого
  прогресса в общем стиле; доступность (aria-live/role, motion-safe); единый маппинг
  ошибок (`lib/errors`) для тостов синхронных отказов.
- **ADR-0018 — E2E и детерминизм воркфлоу**: поллинг статуса без `sleep` (хук
  `useServiceStatus` через `refetchInterval` до терминала).
- **ADR-0004 — guarded-CAS статусов**: клиент только читает статус; переходы и защита от
  двойного запуска — на сервере (CAS→409, показывается тостом).
- **ADR-0005 — Saga-откат**: терминальный `failed` в степпере отражает факт
  компенсирующего отката.
- **ADR-0001/0008 — Temporal/split воркфлоу**: порядок активностей
  `services/projects/{provisioning,changeowners,decommission,transfer}` = справочник
  подписей шагов клиентской модели.
- **ADR-0012/0013 — decommission/transfer**: пометка точки невозврата в степпере.
- **ADR-0019/0020/0021 — реальные GitLab/Vault/Harbor**: их per-activity порядок —
  источник названий шагов (сами плечи не меняются).
