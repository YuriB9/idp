# ADR (изменение: portal-ui-design-system)

Этот файл — индекс архитектурных решений данного изменения. Канонические ADR
лежат в `docs/adr/NNNN-kebab-title.md` (вне `openspec/`).

## Новые ADR

- **ADR-0017 — Дизайн-система и UI-архитектура портала**
  (`docs/adr/0017-portal-design-system-and-ui-architecture.md`): дизайн-система как
  расширение существующих shadcn-примитивов на токенах CSS-переменных + Tailwind;
  собственные лёгкие DataTable (курсорная пагинация ADR-0009), Dialog/ConfirmDialog
  и Toast вместо тяжёлых UI-китов; единый UI-маппинг кодов периметра; ключи
  localStorage для состояния навигации/плотности; доступность (WCAG AA, фокус,
  ARIA, reduced-motion) — требование первого класса.

## Реализуемые существующие ADR

- **ADR-0009 — Форма периметра (REST, keyset/opaque-курсоры, коды ошибок)**:
  DataTable использует курсорную (не offset) пагинацию по
  `next_page_token`/`next_cursor`; единый маппинг покрывает стабилизированные коды
  400/403/404/409/422/503. Контракт не меняется.
- **ADR-0014/0015/0016 — IAM-админка и каталог субъектов**: редизайн раздела «Роли
  и доступы» сохраняет каталог ролей/прав, назначение ролей, пикер пользователя
  (debounce, канонический `sub`) и обработку 403/503/«осиротевших» — переоформление
  без изменения поведения и прав.
