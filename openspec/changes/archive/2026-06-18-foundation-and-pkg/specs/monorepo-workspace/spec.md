## ADDED Requirements

### Requirement: Раскладка модулей go.work

Репозиторий ДОЛЖЕН (MUST) быть монорепо на `go.work` с отдельным Go-модулем для общих пакетов (`./pkg`), по модулю на каждый сервис (`services/gateway`, `services/idm`, `services/projects`, `services/devinfra-worker`) и модулем сквозных тестов (`./tests/e2e`). Все эти модули ДОЛЖНЫ быть перечислены в `go.work`.

#### Scenario: go.work перечисляет все модули монорепо

- **WHEN** разработчик открывает `go.work` в корне репозитория
- **THEN** в нём присутствуют директивы `use` для `./pkg`, каждого `./services/*` и `./tests/e2e`

#### Scenario: Сборка любого модуля проходит через workspace

- **WHEN** выполняется `go build ./...` в корне с активным workspace
- **THEN** все модули собираются с разрешением зависимостей через `go.work` без ручного `replace`

### Requirement: Изолированный модуль инструментов

Модуль `./tools` ДОЛЖЕН (MUST) быть исключён из `go.work` и фиксировать версии инструментов кодогена/линтинга в собственном `go.mod`; его команды ДОЛЖНЫ запускаться с `GOWORK=off`, чтобы tooling-зависимости не протекали в модули сервисов.

#### Scenario: tools не входит в workspace

- **WHEN** проверяется содержимое `go.work`
- **THEN** директива `use ./tools` отсутствует, а `./tools/go.mod` существует и пинит версии инструментов

#### Scenario: Запуск инструмента не задевает граф сервиса

- **WHEN** инструмент из `./tools` запускается с `GOWORK=off`
- **THEN** он использует версии из `./tools/go.mod`, а `go.mod` сервисов не получает новых зависимостей
